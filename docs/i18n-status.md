# KPM 国际化状态记录

更新时间：2026-06-03

## 当前状态

- 当前试点版本默认展示中文。
- 登录页和系统右上角的语言切换按钮已隐藏。
- `prototype-runtime/app.js` 中仍保留部分 `data-i18n`、`translations` 和 `applyLanguage()` 逻辑，但 `currentLanguage` 被固定为 `zh`。
- React 工程中已建立 `/Users/henry/Documents/KPM/apps/frontend/kpm-web/src/i18n` 模块，默认语言为 `zh-CN`，保留英文资源作为未来恢复入口。

## 为什么先隐藏

前期页面和按钮仍在频繁调整，完整中英文词条尚未统一抽取。如果现在保留切换入口，用户会看到部分中文、部分英文的混合状态，体验不如默认中文稳定。

## 未来恢复建议

1. 按页面迁移 React 时，将展示文本统一改为 i18n key。
2. 建立中文、英文两套完整资源文件，避免在组件中硬编码展示文案。
3. 增加一轮英文模式 QA，覆盖菜单、表格、按钮、弹窗、toast、空状态和登录页。
4. QA 通过后再恢复登录页与右上角语言切换按钮。
